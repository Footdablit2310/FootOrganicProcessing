# Versioning rules applied to Foot: Organic Processing
## Versioning scheme
Foot: Organic Processing uses the O.M.S.H versioning scheme(also known as Epoch SemVer but with a few modifications). The version number is in the format of `O.M.S.H` where:
- `O` is the overhaul version, which is incremented when there are breaking changes that are not backward compatible with all the previous version's, like code changes.
- `M` is the major version, which is incremented when there are significant new features or improvements that are backward compatible.
- `S` is the semi-major/big-minor version, which is incremented when there are a lot of changes but doesn't qualify as a major version.
- `H` is the hotfix/patch/minor version, which is incremented when there are urgent bug fixes/minor improvements.
